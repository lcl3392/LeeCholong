- Menu bar -

네비게이션 토글 함수: 페이지가 로드되면 jQuery로 정의된 이벤트 핸들러가 동작합니다. 네비게이션 메뉴를 열고 닫는 행위는 페이지 상단 헤더의 'gnbBtn' 클래스를 가진 요소를 클릭하면 헤더 내비게이션(nav.gnb)에 'on' 클래스를 토글하는 함수가 실행됩니다.

클릭 이벤트 관리: 헤더 버튼 클릭, 본문 클릭, 내부 닫기 버튼 클릭 등 각각의 동작에 대한 클릭 이벤트를 정의하여, 사용자가 메뉴를 편리하게 열고 닫을 수 있도록 구성되어 있습니다.

- 스크롤 애니메이션 -

페이지를 스크롤하거나 창의 크기를 조절할 때 발생하는 이벤트에 따라 여러 가지 효과를 적용하여 보여집니다.

jQuery를 사용하여 화면에 나타나는 요소에 스크롤 애니메이션 효과를 부여합니다.

스크롤 위치 저장: 현재 스크롤 위치를 나타내는 변수 scrollPos를 초기화하고, 스크롤 이벤트가 발생할 때마다 현재 스크롤 위치를 scrollTop() 메서드를 사용하여 업데이트합니다.

스크롤 위치에 따른 효과 적용 - fixHeader() 및 fixList(): 각 함수에서 스크롤 위치에 따라 헤더나 리스트에 특정 클래스를 추가하거나 제거하여 시각적인 변화를 주어 스크롤 애니메이션을 완성합니다.

- 반응형 작업 -

1400px 이상 (대형 화면): 내부 컨테이너와 목록 등의 너비를 90%로 조절하여 콘텐츠의 넓은 화면에 대응합니다. 일부 섹션에서는 특정 요소의 크기와 레이아웃을 조정하여 적절한 시각적 효과를 부여합니다.

1024px (태블릿 등 중간 화면): 제목과 텍스트의 폰트 크기를 조정하고, 섹션 레이아웃을 유연하게 변경하여 사용자에게 더 편한 경험을 제공합니다. 특히 접근성을 고려하여 일부 기능을 개선하고 글꼴 스타일을 조정합니다.

768px (모바일 가로 방향):모바일 가로 화면에서는 일부 섹션의 레이아웃을 세로로 변경하고 이미지 크기를 조절하여 사용자에게 최적화된 뷰를 제공합니다. 특히 저해상도 디바이스에서의 이미지 및 텍스트 렌더링을 고려하여 스타일을 수정합니다.

500px (최소 모바일 화면): 헤더 요소들의 크기와 간격을 최적화하고, 내비게이션 메뉴를 축소하여 더 나은 모바일 사용자 경험을 제공합니다. 특히 일부 섹션에서는 레이아웃을 단순화하고 일부 요소를 감추어 화면을 간소화합니다.

모든 크기에서는 viewport 설정을 통해 화면 크기와 초기 배율을 조절하고, 미디어 쿼리를 활용하여 각 화면 크기에 맞게 스타일을 동적으로 조정하여 반응형 웹 디자인을 완성합니다.
